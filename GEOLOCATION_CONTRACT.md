# Geolocation Contract (VIVO)

## Canonical storage strategy

- **DB canonical type:** `geography(Point, 4326)` for persisted point locations.
- **Frontend canonical shape:** `{ lat: number; lon: number }`.
- **Transport to DB:** WKT `POINT(lon lat)` generated by helper.
- **Read from DB:** parsed through shared helper supporting WKT/EWKT and legacy JSON fallback.

## Implemented helpers

- `src/lib/geospatial.ts`
  - `isValidGeoPoint(point)`
  - `toDbGeographyPoint(point)`
  - `fromDbGeographyPoint(raw)`

## Rules

1. Do not serialize geolocation as ad-hoc JSON in app code.
2. Do not parse PostGIS strings manually in UI components.
3. Use `toDbGeographyPoint` for writes (`drivers.current_location`, `trips.origin_location`, `trips.destination_location`).
4. Use `fromDbGeographyPoint` for reads whenever `geography(Point, 4326)` is selected.

## Migration note

- Existing trip coordinate sync trigger (`trips` lat/lon <-> geography) remains source of truth for consistency.
- Added no extra geolocation columns in this iteration.
